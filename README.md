# HDFC Mutual Fund Purchase Optimizer – Frontend

A modern, responsive frontend application that enables users to assess their risk profile, explore mutual fund recommendations, and visualize optimized allocations generated by the backend decision engine.

The frontend focuses on **clarity, usability, and performance**, acting as the presentation and interaction layer for the Mutual Fund Purchase Optimizer system.

## Logical Architecture

The frontend is designed using **feature-based modular architecture** to ensure scalability and maintainability.

### 1. User Input Layer
- Collects user demographics and investment preferences  
- Risk Profile Questionnaire (Age, Income, Duration, Risk Appetite)  
- Client-side validation and UX feedback  

### 2. API Interaction Layer
- Communicates with the FastAPI backend  
- Handles async states (loading, success, error)  
- Token-based authentication (JWT-ready)  

### 3. Visualization & Results Layer
- Displays optimized fund allocations  
- Risk vs Return summaries  
- Charts and tables for allocation breakdown

## Architecture

The backend follows a **layered architecture** pattern with strict separation of concerns:

```
Client Request
    │
    ▼
┌─────────────────────────────────┐
│  CORS Middleware                │
├─────────────────────────────────┤
│  FastAPI Router (10 modules)    │  ◄── URL matching + Pydantic validation
├─────────────────────────────────┤
│  Dependency Injection           │  ◄── Auth (JWT), DB Session
├─────────────────────────────────┤
│  Services Layer                 │  ◄── Business logic, Risk Engine
├─────────────────────────────────┤
│  SQLAlchemy ORM (16 models)     │  ◄── Data access
├─────────────────────────────────┤
│  PostgreSQL 15 (AWS RDS)        │  ◄── Persistent storage
└─────────────────────────────────┘
```

## Mathematical Model

The recommendation engine uses a **three-layer optimization model**:

### Layer 1: User Risk Scoring
```
S_capacity = 0.4 × (1 - Age/60) + 0.3 × min(Income/2M, 1) + 0.3 × min(Duration/15, 1)
S_stated   = (mean(Q1..Q4) - 1) / 3
S_final    = 0.6 × S_stated + 0.4 × S_capacity        (new users)
           = α × S_stated + (1-α) × S_revealed         (returning users)
```

### Layer 2: Fund Evaluation
```
FRS = 0.4 × NormVolatility + 0.6 × NormDrawdown
FQS = 0.5 × NormAlpha + 0.4 × NormSortino + 0.1 × (1 - NormExpenseRatio)
```

### Layer 3: Allocation
```
FinalScore = W_fqs × FQS - W_rm × |S_final - FRS| + W_df × DurationFit
Allocation = (FinalScore / ΣFinalScores) × TotalInvestment
```

## Getting Started

### Prerequisites
- **Python 3.11+** (minimum required version)
- PostgreSQL 15+
- Docker (optional, recommended)

### 1. Clone and Setup

```bash
git clone https://github.com/oligo434/MF_Purchase_Optimizer_BE.git
cd MF_Purchase_Optimizer_BE

# Create virtual environment
python3 -m venv venv
source venv/bin/activate    # Linux/macOS
# venv\Scripts\activate     # Windows

# Install dependencies
pip install -r requirements.txt
```

### 2. Configure Environment

Create a `.env` file in the project root:

```env
DATABASE_URL="postgresql+psycopg2://user:password@localhost:5432/mfpo"
SECRET_KEY="your-secret-key"
ALGORITHM="HS256"
ACCESS_TOKEN_EXPIRE_MINUTES="30"
REFRESH_TOKEN_EXPIRE_DAYS="7"
ADMIN_SECRET="your-admin-secret"
SMTP_EMAIL="your-email@gmail.com"
SMTP_PASSWORD="your-app-password"
ENCRYPTION_KEY="your-fernet-key"
FRONTEND_URL="http://localhost:5173"
```

### 3. Run Migrations and Start

```bash
# Apply database migrations
alembic upgrade head

# Start the development server
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

The API will be available at `http://localhost:8000` with interactive docs at `/docs`.
