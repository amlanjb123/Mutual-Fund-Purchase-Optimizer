⚖️ Risk Score Engine
User_Risk_Score_Engine.py
Stated Risk = avg(Q1–Q4) ÷ 3
Risk Capacity = Age + Income + Duration
New user: 50% Stated + 50% Capacity
Returning: blends revealed behaviour
→ final_risk_score → Low / Moderate / High
🎯 Recommendation Engine
recommendations.py
Filter funds: |FRS − user_score| ≤ 0.30
Score = 0.5×FQS − 0.3×RiskMismatch + 0.2×DurationFit
Basket size: ₹≤20L→4 funds … >5Cr→8 funds
Waterfall allocation: max 25% per fund
Expected return: FV = amount × (1+CAGR)^years

Role-Specific Flow Steps

🔵 Client (Investor) Flow

①
Register — POST /auth/register
AES-encrypted password · age ≥ 18 · OTP sent to email
OTP emailed via Gmail SMTP (BackgroundTask)
②
Verify Email OTP — POST /auth/verify-otp
10-min TTL · marks User.is_verified = True
Account activated → redirect to login
③
Login + 2FA — POST /auth/login → /auth/verify-login-otp
Returns JWT access token (30 min) + refresh token (7 days)
Tokens stored in localStorage → /client-dashboard
④
Risk Profile Setup — PUT /investor/profile
Fill Q1–Q4, annual income, investment duration
→ Risk Score Engine calculates final_risk_score
⑤
Get Recommendations — POST /recommendations/
Enter amount + duration → AI scores + Waterfall allocation
Rec Engine uses Risk Engine + Fund Catalogue
⑥
Invest — POST /investor/invest (or /invest-all)
Min amount check · units = amount ÷ latest NAV → PENDING
Notification sends order alert email to advisor
⑦
Track Orders & Portfolio
GET /investor/pending-orders · /history · /portfolio (APPROVED only)

🟢 Advisor Flow

Ⓐ
Register with ARN — POST /auth/register (role=advisor)
ARN format validated · OTP email verify (same as client)
Account activated with valid ARN number
Ⓑ
Login + 2FA — POST /auth/login → /auth/verify-login-otp
JWT role=advisor → Guard: role in (advisor | admin)
→ /advisor-dashboard
Ⓒ
View All Clients — GET /advisor/clients
Returns full investor profiles: risk scores, Q answers, DOB, income
Ⓓ
View Client Portfolio — GET /advisor/clients/:id/portfolio
Sees ALL transactions including PENDING and REJECTED
Ⓔ
Review Orders — GET /advisor/orders (PENDING first)
See: client name · fund · amount · AI recommendation reason
Ⓕ
Approve Order — POST /advisor/orders/:id/approve
Status → APPROVED · 2 emails sent (investor + advisor)
Ⓖ
Reject Order — POST /advisor/orders/:id/reject
Status → REJECTED · rejection reason · email sent to investor

🟠 Admin Flow

❶
Register — POST /auth/register-admin
Requires ADMIN_SECRET env var · Auto-verified (no OTP needed)
Account is immediately active, no email OTP
❷
Login + 2FA — POST /auth/login → /auth/verify-login-otp
JWT role=admin → Guard: role must == "admin"
→ /admin-dashboard
❸
View System Stats — GET /admin/stats
Total users, investors, advisors, active funds, last NAV sync
❹
Manage Funds — GET/POST/PUT /admin/funds
Add fund: auto-fetches metadata from mfapi.in
Update expense ratio · category · deactivate (soft delete)
DELETE = soft deactivate (data preserved, excluded from recs)
❺
Trigger NAV Ingestion — POST /admin/ingest/nav
Runs ingest_mf_data.py as BackgroundTask subprocess
Fetches daily NAV from mfapi.in → writes to nav_history table
❻
Trigger Metrics — POST /admin/ingest/metrics
Runs compute_metrics.py → updates FQS · FRS · CAGR · Beta · Alpha
APScheduler also runs both automatically at 01:00 daily
❼
Audit Orders — GET /admin/approved-orders
Full audit log: client · fund · amount · approving advisor name

⚡ Key Cross-Role Interactions

Client ──→ Advisor When client places an investment → Notification Service emails advisor with order details
Advisor ──→ Client When advisor approves or rejects → Notification Service emails investor with status + reason
Admin ──→ Client Fund deactivated by admin → excluded from future recommendations (is_active = False)
Admin ──→ Engines Admin triggers metrics recompute → Risk Engine + Rec Engine get updated FQS/FRS scores
